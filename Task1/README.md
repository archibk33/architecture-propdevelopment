# Задание 1: Проверочный лист по безопасности данных

## Ход работы

### 1. Разобрал данные компании
Изучено, какие данные использует PropDevelopment, и разделил их на 4 группы:
- **Публичные данные** - информация о ЖК, контакты застройщика, новости, цены
- **Внутренние данные** - настройки систем, архитектурные схемы, документация, метрики
- **Конфиденциальные данные** - личные данные клиентов, платежная информация, настройки умного дома
- **Секретные данные** - API ключи, стратегические планы, пароли админки, криптоключи

### 2. Риски
Для каждой группы данных определено возможные проблемы:
- Утечка данных
- Потеря данных  
- Искажение данных
- Некачественные данные
- Обесценивание данных

### 3. Опасность
Поставлены оценки рискам:
- **Критический** - нарушение закона, большие финансовые потери
- **Значительный** - проблемы в работе, но можно исправить
- **Незначительный** - временные сбои

### 4. Диаграмма
Созданы mindmap в draw.io:
- Центральный блок "Безопасность данных"
- 4 группы данных с примерами
- Риски для каждой группы с оценками
- Объяснение, почему я так оценил

## Файлы в папке

- `data_analysis.md` - подробный анализ данных и рисков
- `security_mindmap.drawio` - диаграмма в формате draw.io
- `README.md` - файл с описанием Task 1

## Схемы

1. Открыть файл `security_mindmap.drawio` в draw.io (https://app.diagrams.net/)
2. Скопировать XML-код в любой редактор диаграмм
