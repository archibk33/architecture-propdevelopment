# Анализ данных PropDevelopment на основе стандарта ISO/IEC 27001/27002

## Категории данных

### 1. Публичные данные
Это информация, которую можно показывать всем без ограничений.

**Сюда относится:**
- Информация о жилом комплексе
- Контактные данные застройщика
- Новости и обновления по проекту
- Актуальные цены на жильё

**Какие риски есть:**
- **Обесценивание данных**: Критический - конкуренты могут использовать информацию против нас
- **Искажение данных**: Значительный - если опубликуем неправильную информацию, репутация пострадает

### 2. Внутренние данные
Данные для внутреннего использования, без персональной информации.

**Сюда относится:**
- Настройки и конфиги систем
- Архитектурные схемы и решения
- Документация по техчасти
- Показатели и метрики работы

**Какие риски есть:**
- **Утечка данных**: Значительный - могут раскрыть, как устроена компания изнутри
- **Потеря данных**: Значительный - бизнес-процессы могут сломаться
- **Искажение данных**: Значительный - системы могут работать неправильно

### 3. Конфиденциальные данные
Самое важное - данные клиентов и собственников, которые нужно защищать.

**Сюда относится:**
- Личные данные клиентов
- Платежная информация
- Настройки умного дома
- Инфо по объектам недвижимости
- Данные по ЖКХ услугам
- Информация по совершённым сделкам

**Какие риски есть:**
- **Утечка данных**: Критический - нарушение закона о персональных данных, штрафы, потеря доверия клиентов
- **Потеря данных**: Критический - бизнес может остановиться, финансовые потери
- **Искажение данных**: Критический - ошибки в сделках, проблемы с деньгами
- **Некачественные данные**: Значительный - системы работают с ошибками

### 4. Секретные данные
Самое секретное - ключи, пароли, стратегические планы.

**Сюда относится:**
- Ключи для API-шек
- Стратегические документы и планы
- Пароли от админки
- Коды для доступа к системе безопасности
- Криптоключи (для шифрования и всего такого)

**Какие риски есть:**
- **Утечка данных**: Критический - полная компрометация всех систем безопасности
- **Потеря данных**: Критический - не сможем войти в важные системы
- **Искажение данных**: Критический - системы безопасности перестанут работать

## Почему я так оценил риски

### Критический риск
- **Утечка конфиденциальных данных**: Нарушение 152-ФЗ "О персональных данных", штрафы до 6 млн рублей
- **Утечка секретных данных**: Полная компрометация систем безопасности
- **Потеря конфиденциальных данных**: Бизнес может остановиться, большие финансовые потери
- **Обесценивание публичных данных**: Конкуренты получат преимущество

### Значительный риск
- **Утечка внутренних данных**: Раскрытие внутренней структуры, но не критично
- **Искажение данных**: Проблемы в работе систем, но можно восстановить
- **Потеря внутренних данных**: Нарушение процессов, но можно восстановить

### Незначительный риск
- **Некачественные внутренние данные**: Временные сбои в работе систем

3. Важно централизовать управление данными
