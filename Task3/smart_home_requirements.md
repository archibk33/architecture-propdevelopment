# Требования к внешним интеграциям: сервисы "Умный дом"

## 1. Требования к безопасности
- Все взаимодействия между мобильным приложением, внутренними сервисами и внешней платформой должны быть защищены (TLS/SSL)
- Данные биометрии и номера автомобилей передаются только в зашифрованном виде
- Аудит и логирование всех попыток доступа к сервисам "Умный дом"
- Внешний партнёр должен соответствовать требованиям по защите персональных данных (152-ФЗ, GDPR)

## 2. Протоколы аутентификации и авторизации
- Использовать OAuth 2.0/OpenID Connect (SSO через корпоративный Identity Provider)
- JWT-токены с ограниченным временем жизни для сервисных взаимодействий
- Авторизация по ролям (RBAC) для биометрии и номеров автомобилей

## 3. Организация взаимодействия
- Взаимодействие между мобильным приложением и внешней платформой — через API Gateway PropDevelopment
- Внешний партнёр предоставляет REST API для сервисов домофона и шлагбаума
- Все вызовы внешних API проходят через внутренний шлюз (контроль доступа, аудит, мониторинг)
- В случае сбоя внешней платформы — обработка ошибок и информирование пользователя в приложении
