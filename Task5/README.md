# Task5 - Управление трафиком внутри кластера Kubernetes

## Описание задания

В рамках данного задания реализовано разграничение трафика между сервисами, развернутыми в кластере Kubernetes, с использованием NetworkPolicies. Целью является обеспечение изоляции между пользовательским и административным доменами системы.

## Архитектура сервисов

Система состоит из четырех основных сервисов с соответствующими метками:

1. **front-end** - пользовательский интерфейс
2. **back-end-api** - API для пользователей
3. **admin-front-end** - административный интерфейс
4. **admin-back-end-api** - административное API

## Сетевые политики

Реализованы четыре NetworkPolicy для обеспечения изоляции трафика:

### 1. front-end-isolation
- **Назначение**: Изоляция front-end от других сервисов
- **Разрешенный трафик**: Только с back-end-api на порту 80

### 2. back-end-api-isolation
- **Назначение**: Изоляция back-end-api от других сервисов
- **Разрешенный трафик**: Только с front-end на порту 80

### 3. admin-front-end-isolation
- **Назначение**: Изоляция admin-front-end от других сервисов
- **Разрешенный трафик**: Только с admin-back-end-api на порту 80

### 4. admin-back-end-api-isolation
- **Назначение**: Изоляция admin-back-end-api от других сервисов
- **Разрешенный трафик**: Только с admin-front-end на порту 80

## Развертывание сервисов

Для создания тестовой среды выполните следующие команды:

```bash
# Создание сервисов с соответствующими метками
kubectl run front-end-app --image=nginx --labels role=front-end --expose --port 80
kubectl run back-end-api-app --image=nginx --labels role=back-end-api --expose --port 80
kubectl run admin-front-end-app --image=nginx --labels role=admin-front-end --expose --port 80
kubectl run admin-back-end-api-app --image=nginx --labels role=admin-back-end-api --expose --port 80
```

## Применение сетевых политик

После создания сервисов примените сетевые политики:

```bash
kubectl apply -f network_policies.yaml
```

## Тестирование изоляции

### Разрешенные соединения
- front-end ↔ back-end-api
- admin-front-end ↔ admin-back-end-api

### Заблокированные соединения
- front-end ↔ admin-back-end-api
- admin-front-end ↔ front-end
- Любые другие междоменные соединения

## Принципы безопасности

1. **Сегментация сети**: Четкое разделение пользовательского и административного трафика
2. **Принцип наименьших привилегий**: Каждый сервис имеет доступ только к необходимым ресурсам
3. **Изоляция доменов**: Полная изоляция между пользовательскими и административными сервисами
4. **Защита от горизонтального перемещения**: Блокировка несанкционированного доступа между сервисами

## Структура файлов

- `network_policies.yaml` - конфигурация всех сетевых политик
- `README.md` - данная документация
- `cleanup.bat` - скрипт очистки ресурсов

## Очистка ресурсов

Для удаления всех созданных ресурсов используйте:

```bash
./cleanup.bat
```

## Заключение

Реализованная система NetworkPolicies обеспечивает надежную изоляцию трафика между сервисами в кластере Kubernetes. Достигнута полная сегментация пользовательского и административного доменов, что соответствует принципам безопасности современных облачных приложений.

Система готова к использованию в продакшн среде с дополнительной настройкой мониторинга и логирования сетевого трафика.
