# Task 4: Защита доступа к кластеру Kubernetes

## Описание задания

Необходимо организовать ролевой доступ к Kubernetes для пользователей кластера PropDevelopment на основе анализа безопасности из предыдущих заданий.

## Контекст из предыдущих заданий

На основе анализа Task1-Task3 выявлены следующие ключевые проблемы:
- Множественные точки аутентификации
- Отсутствие централизованного управления доступом
- Недостаточная защита критичных данных
- Отсутствие разграничения доступа по организационной структуре

## Структура файлов

- `roles_table.md` - таблица ролей и полномочий
- `create_users.sh` - скрипт создания пользователей
- `create_roles.sh` - скрипт создания ролей
- `bind_users_to_roles.sh` - скрипт связывания пользователей с ролями
- `README.md` - данный файл

## Организационная структура PropDevelopment

На основе анализа архитектуры выделены следующие группы:
1. **Администраторы безопасности** - полный доступ к кластеру
2. **DevOps инженеры** - настройка и развертывание
3. **Разработчики** - просмотр и мониторинг
4. **Операционные менеджеры** - ограниченный просмотр

## Требования безопасности

- Принцип наименьших привилегий
- Разграничение доступа по доменам (Продажи, ЖКУ, Финансы, Данные)
- Аудит всех действий с критичными ресурсами
- Защита секретов и конфигураций
